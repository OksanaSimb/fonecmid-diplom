# Инструкция по начальной настройке подсистемы "Сервисное обслуживание и Расчеты"
## Версия 1.0 (Релиз от 04.07.2026)

### 1. Введение
Настоящий документ описывает порядок действий для перехода к работе в обновленной конфигурации, включающей функционал:
- Абонентское обслуживание (новый вид договоров);
- Учет заявок и обслуживания клиентов;
- Расчет управленческой зарплаты и отпусков;
- Автоматическое формирование актов и интеграция с Telegram.

**Внимание!** Все настройки выполняются в **рабочей базе**, где уже ведется учет. 
Перед началом убедитесь, что у вас есть права администратора.


### 2. Заполнение констант (Обязательный минимум)
Перед запуском механизмов необходимо задать базовые параметры учета и интеграции.

| **НоменклатураАбонентскаяПлата** | Услуга для ежемесячной абон. платы | Обслуживание клиентов -> Абонентская плата| Выбрать элемент справочника «Номенклатура» (создайте новый, например, «Абонентская плата») |
| **НоменклатураРаботыСпециалиста** | Услуга для почасовой работы | Обслуживание клиентов -> Работы специалиста| Выбрать элемент (например, «Работы специалиста») |
| **ТокенУправленияТелеграмБотом** | Для интеграции с ботом | Обслуживание клиентов -> Токен управления телеграм ботом | Вставить токен бота |
| **ИдентификаторГруппыДляОповещения** | Куда слать уведомления | Обслуживание клиентов -> Идентификатор группы для оповещения | Вставить числовой ID чата/группы |

### 3. Настройка справочников и нормативно-справочной информации

#### 3.1. Активация нового вида договоров
Для корректной работы заявок необходимо переключить существующие договоры на новый вид.

1. Откройте справочник **«Договоры контрагентов»**.
2. Выберите договор с клиентом.
3. В поле **«Вид договора»** выберите значение **«Абонентское обслуживание»**.
4. Заполните появившиеся реквизиты:
   - *Дата начала договора*;
   - *Дата окончания договора*;
   - *Абонентская плата *;
   - *Стоимость работы специалиста*.
5. Нажмите «Записать».

> *Важно:* Договор будет активен для проведения документов обслуживания, если текущая дата попадает в указанный период.

#### 3.2. Установка окладов и процентов для сотрудников
Для расчета зарплаты (специалистам) необходимо задать условия оплаты:

1. Перейдите в раздел «Расчет управленческой зарплаты» -> **«Условия оплаты сотрудников»**.
2. На каждый месяц создайте запись для каждого сотрудника:
   - *Сотрудник* (выберите из справочника);
   - *Оклад* (сумма);
   - *Процент от работ* (например, 10, если специалист получает 10% от суммы выставленных клиенту работ).


### 4. Настройка прав доступа (Назначение ролей и профилей)
В системе появились новые бизнес-роли. Необходимо назначить их пользователям.

**Для этого перейдите:** Администрирование -> Настройка пользователей и прав -> Профили групп доступа.

| Профиль | Входящие роли (новые) | Кто получает |

| **Специалист** | Роль "ОбслуживаниеКлиентов" + "ОтчетыВыработка" | Инженеры, мастера (для ведения заявок) |

| **Менеджер ИТ-фирмы** | Роль "ОбслуживаниеКлиентов" + "ПросмотрЗарплаты" | Менеджеры по работе с клиентами |

| **Бухгалтер ИТ Фирмы** | Роль "ЗаполнениеРеализаций" + "ОтчетыАнализАктов" | Бухгалтер (для массового формирования актов) |

| **Кадровик-расчетчик** | Роль "НачислениеЗарплаты" + "ГрафикОтпусков" + "ОтчетыЗарплата" | Бухгалтер по зарплате / Кадровик |

### 5. Инициализация вспомогательных объектов (Telegram-бот)
Механизм оповещений использует справочник-очередь. Для его работы выполните одноразовое действие:

1. Откройте раздел «Администрирование» -> Обслуживание  ->  «Регламентные и фоновые задания».
2. Убедитесь, что задание **«Отправка сообщений Telegram»** включено и активен планировщик (например, запуск каждые 5 минут).
3. Вручную запустите задание для проверки соединения с ботом.


### 6. Ввод начальных остатков (если переход с 1 числа месяца)
Если на момент внедрения у вас уже есть текущие договоры:

1. Зайдите в документ **«Обслуживание клиента»**.
2. Создайте документы за прошедшие периоды (например, за прошлый месяц) с пометкой «Проведен», чтобы регистры накопления сразу наполнились историей для расчета зарплаты и построения отчетов.
